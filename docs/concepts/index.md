## 核心理念

### 分层架构

ZZFrame 采用经典的分层架构，每一层都有明确的职责：

```
┌─────────────────┐
│   Controller    │  HTTP 请求处理、参数验证
├─────────────────┤
│     Service     │  业务逻辑、事务管理
├─────────────────┤
│       DAO       │  数据访问、SQL 查询
├─────────────────┤
│    Database     │  数据持久化
└─────────────────┘
```

### 约定优于配置

框架采用约定优于配置的设计理念，提供合理的默认值，减少配置工作：

- 默认的目录结构
- 默认的命名规范
- 默认的配置值

### 开箱即用

框架内置了企业级应用常用的功能：

- 用户管理
- 角色管理
- 菜单管理
- 登录日志
- 权限控制

### 可扩展性

框架设计时考虑了可扩展性，你可以：

- 自定义中间件
- 扩展服务层
- 自定义验证规则
- 集成第三方组件

## 技术栈

ZZFrame 基于 GoFrame 框架构建，集成了以下技术：

- **GoFrame**: 高性能 Web 框架
- **GORM**: ORM 库（可选）
- **Casbin**: 权限控制
- **Redis**: 缓存
- **MySQL**: 数据库

## 设计模式

框架使用了多种设计模式：

- **单例模式**: Service 层使用单例
- **工厂模式**: DAO 层使用工厂创建
- **依赖注入**: Controller 注入 Service
- **策略模式**: 缓存适配器使用策略模式
